# Plan 03-01: Recipes Backend

## Goal
Recipe entity with ingredients, steps, and optional photo upload. Meal planner assigns recipes to days. Full REST API secured by Spring Security.

## Scope
- Recipe entity: title, description, servings, prep/cook time, photo URL
- Ingredients: ordered list (name, amount, unit)
- Steps: ordered list with text
- MealPlan: date + recipe reference
- File upload for recipe photos (store in /uploads or Railway volume)
- REST endpoints: CRUD recipes, CRUD meal plan
- Search recipes by title (case-insensitive)

## Out of Scope
- Frontend (03-02, 03-03)
- Nutrition data
- External recipe import

## Implementation

### 1. Flyway: V3__create_recipes.sql
```sql
CREATE TYPE meal_slot AS ENUM ('BREAKFAST', 'LUNCH', 'DINNER', 'SNACK');

CREATE TABLE recipes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(255) NOT NULL,
    description TEXT,
    servings INT,
    prep_minutes INT,
    cook_minutes INT,
    photo_url VARCHAR(512),
    created_by UUID NOT NULL REFERENCES users(id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE recipe_ingredients (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    recipe_id UUID NOT NULL REFERENCES recipes(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    amount DECIMAL(10,3),
    unit VARCHAR(50),
    position INT NOT NULL DEFAULT 0
);

CREATE TABLE recipe_steps (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    recipe_id UUID NOT NULL REFERENCES recipes(id) ON DELETE CASCADE,
    text TEXT NOT NULL,
    position INT NOT NULL
);

CREATE TABLE meal_plan (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    plan_date DATE NOT NULL,
    slot meal_slot NOT NULL DEFAULT 'DINNER',
    recipe_id UUID NOT NULL REFERENCES recipes(id) ON DELETE CASCADE,
    notes VARCHAR(512),
    UNIQUE(plan_date, slot)
);

CREATE INDEX idx_recipes_title ON recipes(lower(title));
CREATE INDEX idx_meal_plan_date ON meal_plan(plan_date);
```

### 2. Models
- `Recipe.java` ‚Äî @Entity, OneToMany ingredients + steps (cascade ALL, orphanRemoval), ManyToOne createdBy
- `RecipeIngredient.java` ‚Äî @Entity, ManyToOne recipe, name, amount, unit, position
- `RecipeStep.java` ‚Äî @Entity, ManyToOne recipe, text, position
- `MealPlan.java` ‚Äî @Entity, planDate (LocalDate), slot (MealSlot enum), ManyToOne recipe

### 3. Repositories
- `RecipeRepository` ‚Äî findByTitleContainingIgnoreCase, findAllByOrderByTitleAsc
- `MealPlanRepository` ‚Äî findByPlanDateBetweenOrderByPlanDateAscSlotAsc

### 4. DTOs (Java records)
- `RecipeRequest` ‚Äî title, description, servings, prepMinutes, cookMinutes, ingredients, steps
- `IngredientRequest` ‚Äî name, amount, unit, position
- `StepRequest` ‚Äî text, position
- `RecipeResponse` ‚Äî with static from(Recipe)
- `MealPlanRequest` ‚Äî planDate, slot, recipeId, notes
- `MealPlanResponse` ‚Äî with static from(MealPlan)

### 5. Services
- `RecipeService` ‚Äî createRecipe, updateRecipe, deleteRecipe, getAll, getById, search
- `MealPlanService` ‚Äî setMealPlan (upsert), getWeek(LocalDate from, LocalDate to), removeMealPlan
- Photo upload: `PhotoService` ‚Äî save to `uploads/` directory, return relative URL

### 6. Controllers
- `RecipeController` ‚Äî GET /api/recipes, GET /api/recipes/search?q=, GET /api/recipes/{id}, POST, PUT, DELETE
- `MealPlanController` ‚Äî GET /api/meal-plan?from=&to=, PUT /api/meal-plan/{date}/{slot}, DELETE /api/meal-plan/{date}/{slot}
- `PhotoController` ‚Äî POST /api/recipes/{id}/photo (multipart), serves static /uploads/**

### 7. Security
- All recipe endpoints: any authenticated user (family = shared recipe book)
- Only creator or ADMIN can update/delete a recipe
