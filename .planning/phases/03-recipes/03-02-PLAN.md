# Plan 03-02: Recipes Frontend — List, Detail, Create/Edit

## Goal
Recipe browsing, detail view, and create/edit form. Uses the same patterns established in Phase 2 Tasks frontend.

## Tech Stack Reminders
- Next.js 16.2.1 (NOT 15 — has breaking changes)
- React 19 with `useActionState` for forms (NOT react-hook-form)
- Tailwind CSS v4 (CSS-first, no tailwind.config.js)
- Access token in module-level memory variable (NOT localStorage)
- `apiFetch` from `frontend/lib/api.ts` for all API calls

## Scope
- `frontend/lib/recipes.ts` — typed API functions
- Recipe list page with search
- Recipe detail page with ingredients + steps
- Create recipe form
- Edit recipe form (reuse same component)
- Photo upload on create/edit
- Mobile-friendly card layout

## Out of Scope
- Meal planner (03-03)

## Implementation

### 1. `frontend/lib/recipes.ts`
Types: `Recipe`, `RecipeIngredient`, `RecipeStep`, `RecipeRequest`
Functions:
- `getRecipes(): Promise<Recipe[]>`
- `searchRecipes(q: string): Promise<Recipe[]>`
- `getRecipe(id: string): Promise<Recipe>`
- `createRecipe(data: RecipeRequest): Promise<Recipe>`
- `updateRecipe(id: string, data: RecipeRequest): Promise<Recipe>`
- `deleteRecipe(id: string): Promise<void>`
- `uploadPhoto(id: string, file: File): Promise<{ photoUrl: string }>`

### 2. `frontend/components/recipes/RecipeCard.tsx`
- Photo thumbnail (with fallback placeholder)
- Title, prep+cook time, servings
- Created by initials badge
- Link to detail page

### 3. `frontend/components/recipes/IngredientEditor.tsx`
- Dynamic add/remove ingredients
- Fields: name (text), amount (number), unit (select: g/kg/ml/l/tbsp/tsp/pcs/handful/pinch)
- Enter key adds new row

### 4. `frontend/components/recipes/StepEditor.tsx`
- Dynamic add/remove steps
- Numbered automatically
- Textarea per step
- Drag hint (visual only, no actual drag-drop)

### 5. `frontend/components/recipes/RecipeForm.tsx`
- Fields: title, description, servings, prepMinutes, cookMinutes
- IngredientEditor embedded
- StepEditor embedded
- Photo upload input (shows preview)
- `useActionState` for form submission
- Works for both create and edit (pass existing recipe or null)

### 6. `frontend/app/(app)/recipes/page.tsx`
- Search bar (debounced, 300ms)
- Grid of RecipeCard components
- "Add Recipe" button → /recipes/new
- Empty state with friendly message

### 7. `frontend/app/(app)/recipes/[id]/page.tsx`
- Full recipe display: photo header, metadata row (time, servings)
- Ingredients list with amounts
- Numbered steps
- Edit button (creator or admin only)
- Delete button with confirm (creator or admin only)

### 8. `frontend/app/(app)/recipes/new/page.tsx`
- Wraps RecipeForm, redirects to detail on success

### 9. `frontend/app/(app)/recipes/[id]/edit/page.tsx`
- Fetches recipe, passes to RecipeForm, redirects to detail on success

## Success Criteria
1. Recipe list loads and displays cards with photo/fallback
2. Search filters recipes in real time
3. Detail page shows all ingredients and steps
4. Create form submits and redirects to new recipe
5. Edit form pre-fills and saves changes
6. Delete with confirm works
