# Plan 04-01: Travel Backend

## Goal
Trip planning backend: trips with dates/destination, shared + per-person packing lists, itinerary with activities, and key info storage (hotel, flights, etc.).

## Scope
- Trip entity: title, destination, start/end date, description, cover photo
- Packing list items: shared (visible to all) + per-person (each member's own)
- Itinerary entries: date, time, title, description, location
- Key info entries: label + value (e.g. "Hotel" → "Marriott Berlin, booking #123")
- REST API for all of the above
- Any family member can create/edit trips
- Only trip creator or ADMIN can delete a trip

## Out of Scope
- Frontend (04-02, 04-03)
- External booking integrations
- Budget tracking

## Implementation

### 1. Flyway: V4__create_travel.sql
```sql
CREATE TABLE trips (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(255) NOT NULL,
    destination VARCHAR(255) NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    description TEXT,
    cover_photo_url VARCHAR(512),
    created_by UUID NOT NULL REFERENCES users(id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE packing_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id UUID NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    owner_id UUID REFERENCES users(id) ON DELETE CASCADE,  -- NULL = shared
    name VARCHAR(255) NOT NULL,
    packed BOOLEAN NOT NULL DEFAULT false,
    position INT NOT NULL DEFAULT 0
);

CREATE TABLE itinerary_entries (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id UUID NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    entry_date DATE NOT NULL,
    entry_time TIME,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    location VARCHAR(255),
    position INT NOT NULL DEFAULT 0
);

CREATE TABLE trip_key_info (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id UUID NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    label VARCHAR(100) NOT NULL,
    value TEXT NOT NULL,
    position INT NOT NULL DEFAULT 0
);

CREATE INDEX idx_trips_start_date ON trips(start_date);
CREATE INDEX idx_packing_items_trip ON packing_items(trip_id);
CREATE INDEX idx_itinerary_trip_date ON itinerary_entries(trip_id, entry_date);
```

### 2. Models
- `Trip.java` — @Entity, OneToMany packingItems (shared only via query), OneToMany itineraryEntries, OneToMany keyInfos (all cascade ALL orphanRemoval), ManyToOne createdBy, LocalDate startDate/endDate
- `PackingItem.java` — @Entity, ManyToOne trip, ManyToOne owner (nullable = shared), name, packed boolean, position
- `ItineraryEntry.java` — @Entity, ManyToOne trip, entryDate LocalDate, entryTime LocalTime (nullable), title, description, location, position
- `TripKeyInfo.java` — @Entity, ManyToOne trip, label, value, position

### 3. Repositories
- `TripRepository` — findAllByOrderByStartDateDesc
- `PackingItemRepository` — findByTripIdAndOwnerIdIsNullOrderByPosition (shared), findByTripIdAndOwnerIdOrderByPosition (personal)
- `ItineraryEntryRepository` — findByTripIdOrderByEntryDateAscEntryTimeAscPositionAsc

### 4. DTOs (Java records)
- `TripRequest` — title, destination, startDate, endDate, description
- `TripResponse` — all fields + keyInfos list; static from(Trip)
- `PackingItemRequest` — name, position, ownerId (null = shared)
- `PackingItemResponse` — id, name, packed, position, ownerId, ownerName; static from(PackingItem)
- `ItineraryEntryRequest` — entryDate, entryTime, title, description, location, position
- `ItineraryEntryResponse` — all fields; static from(ItineraryEntry)
- `TripKeyInfoRequest` — label, value, position
- `TripKeyInfoResponse` — id, label, value, position; static from(TripKeyInfo)

### 5. Services
- `TripService` — createTrip, updateTrip, deleteTrip (owner/ADMIN only), getAll, getById, updateCoverPhoto
- `PackingItemService` — addItem, togglePacked, deleteItem, getSharedItems(tripId), getPersonalItems(tripId, userId)
- `ItineraryService` — addEntry, updateEntry, deleteEntry, getEntriesByTrip
- Note: keyInfos managed directly via TripService (part of Trip aggregate)

### 6. Controllers
- `TripController` — GET /api/trips, GET /api/trips/{id}, POST, PUT, DELETE, POST /{id}/photo
- `PackingItemController` — GET /api/trips/{tripId}/packing?personal=false, POST, PATCH /{itemId}/toggle, DELETE /{itemId}
- `ItineraryController` — GET /api/trips/{tripId}/itinerary, POST, PUT /{entryId}, DELETE /{entryId}

## Success Criteria
1. POST /api/trips creates a trip
2. GET /api/trips returns all trips ordered by start date
3. GET /api/trips/{tripId}/packing returns shared packing items
4. GET /api/trips/{tripId}/packing?personal=true returns current user's personal items
5. PATCH /api/trips/{tripId}/packing/{itemId}/toggle flips packed status
6. GET /api/trips/{tripId}/itinerary returns entries ordered by date/time
