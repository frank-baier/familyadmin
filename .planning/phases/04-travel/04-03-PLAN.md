# Plan 04-03: Travel Frontend — Itinerary + Key Info Panel

## Goal
Itinerary view with day-by-day activities and an editable key info panel. Integrates into the trip detail page from 04-02.

## Tech Stack Reminders
- Next.js 16.2.1, React 19, Tailwind CSS v4
- `useActionState` for forms, `apiFetch` for API calls
- No react-hook-form, no localStorage

## Scope
- Add itinerary API functions to `frontend/lib/travel.ts`
- `ItineraryEntry` component — single entry display
- `ItineraryEditor` component — add/edit entries inline
- `ItineraryView` component — day-grouped list of entries
- `KeyInfoPanel` component — display + inline add key info items
- Wire itinerary tab in trip detail page (replace "coming soon" placeholder)
- Add key info management to trip detail

## Out of Scope
- Map integration
- Booking imports

## Implementation

### 1. Add to `frontend/lib/travel.ts`
Functions:
- `getItinerary(tripId: string): Promise<ItineraryEntry[]>`
- `addItineraryEntry(tripId: string, data: ItineraryEntryRequest): Promise<ItineraryEntry>`
- `updateItineraryEntry(tripId: string, entryId: string, data: ItineraryEntryRequest): Promise<ItineraryEntry>`
- `deleteItineraryEntry(tripId: string, entryId: string): Promise<void>`

Types:
- `ItineraryEntryRequest` — entryDate, entryTime (optional), title, description (optional), location (optional), position

### 2. `frontend/components/travel/ItineraryEntry.tsx`
- Time badge (or "All day" if no time)
- Title + location
- Description (collapsed by default, expand on click)
- Edit + Delete buttons (icon buttons)

### 3. `frontend/components/travel/ItineraryEntryForm.tsx`
- Inline form (not a separate page)
- Fields: date, time (optional), title, location (optional), description (optional)
- `useActionState` for submit
- Used for both add and edit

### 4. `frontend/components/travel/ItineraryView.tsx`
- Props: tripId, tripStartDate, tripEndDate
- Loads entries on mount
- Groups entries by date (e.g. "Monday, 23 Jul")
- Each group: date header + list of ItineraryEntry components
- "Add entry" button at the bottom of each day group
- Shows ItineraryEntryForm inline when adding/editing

### 5. `frontend/components/travel/KeyInfoPanel.tsx`
- Props: keyInfos (from trip), tripId (for add API)
- Displays label/value pairs as a clean table/list
- "Add info" button → inline form (label + value text inputs)
- No edit/delete for now (keep simple)

### 6. Modify `frontend/app/(app)/travel/[id]/page.tsx`
- Replace itinerary "coming soon" with `<ItineraryView>`
- Replace static key info display with `<KeyInfoPanel>`

## Files to Create
- `frontend/components/travel/ItineraryEntry.tsx`
- `frontend/components/travel/ItineraryEntryForm.tsx`
- `frontend/components/travel/ItineraryView.tsx`
- `frontend/components/travel/KeyInfoPanel.tsx`

## Files to Modify
- `frontend/lib/travel.ts` — add itinerary functions + types
- `frontend/app/(app)/travel/[id]/page.tsx` — wire in real components

## Success Criteria
1. Itinerary tab shows entries grouped by day
2. "Add entry" form appears inline and submits without page reload
3. Entries can be deleted
4. Key info panel shows trip.keyInfos and allows adding new entries
5. Day headers show formatted date labels
